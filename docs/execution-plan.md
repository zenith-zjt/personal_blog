# 个人博客知识库系统执行计划

## 1. 执行目标

本计划用于把 PDR 落成可开发、可测试、可交付的阶段性任务清单。技术基线采用 `Next.js 16 App Router`，开发时需遵循本地 `node_modules/next/dist/docs/` 文档。

## 2. 总体实施策略

实施顺序遵循：

1. 先定内容模型与目录规则。
2. 再实现文件树与文章读取。
3. 再建设前台浏览体验。
4. 最后补齐后台登录、上传和维护能力。

原因：
- 本项目的核心不是页面壳子，而是“文件即内容源”的内容组织能力。
- 只有内容树规则先稳定，前后台的展示和管理逻辑才不会反复返工。

## 3. 阶段划分

## 阶段 0：方案冻结

目标：
- 冻结信息架构、目录规则、上传规则、后台访问策略。

输出物：
- `docs/pdr-personal-blog.md`
- `docs/design-spec.md`
- 项目级 `AGENTS.md`

完成标志：
- 产品范围明确。
- 知识库目录规则明确。
- 隐藏后台和单管理员约束明确。

## 阶段 1：内容域与文件系统能力

目标：
- 建立知识库根目录与文章解析规则。

任务：
- 约定 `content/` 为知识内容根目录。
- 约定一级目录为知识库。
- 定义 Markdown 文件扫描规则。
- 定义 `resource` 目录排除规则。
- 定义 Markdown 图片解析规则。
- 定义知识库树数据结构。

交付结果：
- 能稳定产出知识库列表。
- 能稳定产出目录树。
- 能从路径定位到文章与资源文件。

风险点：
- 图片路径约定不统一。
- 文件命名重复导致 slug 冲突。

## 阶段 2：前台知识库浏览

目标：
- 完成前台公开浏览闭环。

任务：
- 实现知识库首页。
- 实现知识库详情布局。
- 实现左侧树形导航。
- 实现文章详情渲染。
- 实现当前节点高亮与面包屑。
- 实现移动端导航折叠方案。

交付结果：
- 无需登录即可浏览知识库与文章。
- 目录树可展开、折叠、定位当前文章。

风险点：
- 深层目录在移动端的可用性。
- 大目录下导航性能。

## 阶段 3：搜索能力

目标：
- 提供单篇文章快速检索能力。

任务：
- 定义搜索索引字段：标题、文件名、正文摘要、路径、知识库。
- 实现搜索输入与结果面板。
- 实现结果跳转与关键字段展示。
- 设计无结果、加载中、空状态反馈。

交付结果：
- 用户可通过搜索快速定位文章。

风险点：
- 初版全文扫描在内容量大时性能不足。
- 中文分词策略可能需要后续增强。

## 阶段 4：后台访问控制

目标：
- 完成隐藏后台与单管理员登录能力。

任务：
- 设计隐藏后台路由。
- 实现登录表单。
- 实现单管理员身份校验。
- 实现会话维持与退出。
- 实现未登录访问后台时的拦截。

交付结果：
- 仅管理员可访问后台页面。
- 前台不暴露后台入口。

风险点：
- 默认密码风险。
- 需要明确首次部署后的密码修改流程。

## 阶段 5：后台内容管理

目标：
- 完成上传与结构查看能力。

任务：
- 实现知识库树查看页。
- 实现文章上传页。
- 支持上传 `.md` 文件。
- 支持上传图片资源。
- 支持按目标目录落盘。
- 实现文件类型与命名校验。
- 上传后刷新树数据。

交付结果：
- 管理员可维护内容，无需编辑器。

风险点：
- 文件覆盖策略。
- 同名资源冲突策略。

## 阶段 6：管理员信息维护

目标：
- 提供账号自维护能力。

任务：
- 实现用户名修改。
- 实现密码修改。
- 实现旧密码校验。
- 实现强密码校验。

交付结果：
- 默认管理员可在后台完成凭证替换。

## 阶段 7：质量与上线准备

目标：
- 完成可交付前的质量补齐。

任务：
- 完成关键流程测试。
- 完成异常态测试。
- 完成 SEO 基础配置。
- 补齐部署说明与环境变量说明。
- 检查后台路由是否被任何公开入口泄漏。

交付结果：
- 具备部署上线条件。

## 4. 推荐开发拆分

建议按以下模块拆分：
- 内容扫描模块：负责知识库列表、树结构、文章路径解析。
- Markdown 渲染模块：负责正文展示与图片映射。
- 前台导航模块：负责知识库树和搜索交互。
- 管理认证模块：负责单管理员登录和会话。
- 上传管理模块：负责文件上传、校验、落盘。

## 5. 里程碑

建议里程碑：

1. M1：内容树与文章读取可跑通。
2. M2：前台浏览与详情页可用。
3. M3：搜索可用。
4. M4：后台登录与上传可用。
5. M5：账号修改、测试与上线准备完成。

## 6. 测试计划

关键测试场景：
- 首页知识库列表展示正确。
- 左侧树不显示 `resource`。
- 文章中的图片能从同级 `resource` 正常读取。
- 搜索能定位目标文章。
- 后台未登录不可访问。
- 默认账号可登录后台。
- 用户名密码修改后旧凭证失效。
- 上传 `.md` 与图片后前后台内容结构一致。
- 后台树可见资源目录，前台树不可见资源目录。

## 7. 决策优先级

若开发过程中出现取舍，优先级如下：

1. 内容结构正确性。
2. 后台安全性。
3. 前台浏览效率。
4. 搜索体验。
5. 视觉细节增强。

## 8. 后续扩展预留

后续可扩展但不在本期实施：
- 文章标签与专题页。
- 草稿状态。
- 多知识库封面配置。
- 批量导入与增量同步。
- 全文索引升级。
- 管理员二次验证。
