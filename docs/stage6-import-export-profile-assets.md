# 阶段 6 补充：头像资源与导入导出

## 目标

在账号资料功能可用的基础上，补齐头像文件上传、系统备份和博客迁移能力，使后台可以通过浏览器完成数据归档、恢复与知识库增量迁移。

## 头像资源

- 后台账号设置的资料页支持上传头像文件。
- 头像文件保存到 `data/resource/`。
- 账号配置中只保存相对路径，例如 `resource/avatar.png`。
- 前台首页通过内部资源接口读取头像，避免直接暴露本地文件系统路径。
- 头像格式与知识库图片上传保持一致：`png`、`jpg`、`jpeg`、`gif`、`webp`、`svg`。

## 导入导出

- 后台新增 `导入导出` 标签页。
- 子标签包括 `系统备份` 与 `博客迁移`。
- 系统备份导出范围为 `data/`，通过 ZIP 下载。
- 系统备份导入上传 ZIP，写入 `data/` 前校验 `admin-account.json` 结构是否合法。
- 系统备份导入前提示覆盖风险；如果用户未确认且 `data/` 已有文件，服务端拒绝导入。
- 博客迁移导出范围为 `content/`，通过 ZIP 下载。
- 博客迁移导入为增量写入，同名文件覆盖。
- 博客迁移导入时，普通文章目录只接收 Markdown 与排序配置文件；`resource/` 目录只接收合法图片文件。

## 测试要求

- Playwright 覆盖后台资料页头像上传后前台头像展示。
- Playwright 覆盖后台导入导出页面访问与系统备份、博客迁移包下载主流程。
- 构建与 lint 需要通过，确保新增 Route Handler 与 Server Action 类型合法。
