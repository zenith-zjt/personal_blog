# 个人博客知识库系统 PDR

## 1. 文档目标

本文档定义个人博客知识库系统的产品目标、范围、核心能力、信息结构与验收标准，用于指导后续设计与开发实施。

## 2. 产品定位

该系统不是传统时间流博客，而是“博客 + 知识库”混合产品。

核心定位：
- 对外提供无需登录的公开知识浏览体验。
- 以“知识库”作为一级组织维度，参考语雀的文档集合体验。
- 左侧导航采用类似 IDE 文件树的层级结构，而不是单纯标签列表。
- 支持直接搜索单篇文章，提升快速检索效率。
- 对内提供隐藏式后台，仅由一个管理员维护内容。

## 3. 目标用户

主要用户：
- 博客访客：浏览知识库、阅读文章、搜索内容。
- 管理员：上传 Markdown 与图片、维护知识库目录、修改后台账号信息。

## 4. 业务目标

- 建立统一的个人知识输出入口。
- 兼顾“成体系知识归档”和“单篇文章检索”两种访问方式。
- 降低内容维护成本，避免在线富文本编辑器带来的复杂性。
- 让文章内容直接来源于本地 Markdown 文件，便于迁移、备份与版本管理。

## 5. 产品范围

本期包含：
- 前台知识库首页与文章详情页。
- 左侧知识库树形导航。
- 全局文章搜索能力。
- 后台登录页。
- 后台文章上传页。
- 后台知识库树结构查看页。
- 后台管理员用户名和密码修改页。
- Markdown 图片同级 `resource` 目录解析规则。

本期不包含：
- 前台用户注册、登录、评论、点赞、收藏。
- 多管理员、多角色、多租户。
- 在线 Markdown 编辑器。
- 富文本可视化排版工具。
- 文章协同编辑。
- 公共后台入口暴露。

## 6. 关键产品原则

### 6.1 前台公开，后台隐藏

- 前台所有文章可直接访问，无需登录。
- 后台页面不在前台导航、页脚、公开文案中暴露。
- 后台入口使用独立隐藏路径，不与前台交叉展示。

### 6.2 内容以文件为中心

- 上传方式为 `md` 文件与图片文件上传。
- 不建设文章编辑页。
- 系统以文件结构生成知识库结构，减少额外维护动作。

### 6.3 知识库优先于时间流

- 首页默认展示知识库集合，而非按时间倒序文章流。
- 用户首先进入知识库维度，再进入树形目录，再打开文章。

### 6.4 导航与内容解耦

- 导航展示目录与文章节点。
- `resource` 文件夹参与图片资源解析，但不参与前端树展示。

## 7. 功能需求

## 7.1 前台展示

### 7.1.1 知识库首页

- 展示所有知识库集合。
- 每个知识库展示名称、简介、文章数量、最近更新时间。
- 默认支持桌面端左右布局，移动端折叠为抽屉式导航。

### 7.1.2 知识库树形导航

- 左侧展示当前知识库目录树。
- 目录层级应接近 IDE 文件树体验。
- 支持展开/折叠目录。
- 支持高亮当前文章节点。
- 支持默认展开当前文章所在路径。
- 排除名称为 `resource` 的文件夹。

### 7.1.3 文章详情展示

- 支持渲染 Markdown 正文。
- 支持标题、段落、代码块、引用、表格、列表、图片等常见 Markdown 元素。
- 支持文章基础元信息展示：标题、更新时间、所属知识库、所属路径。
- 图片默认按文章所在目录下同级 `resource` 文件夹解析。

示例：
- 文章路径：`/content/java/spring/intro.md`
- 图片路径：`/content/java/spring/resource/arch.png`
- Markdown 内图片引用按相对约定自动映射至同级 `resource`

### 7.1.4 搜索

- 支持全局搜索单篇文章。
- 支持按标题、文件名、正文关键词检索。
- 支持搜索结果直接跳转文章详情。
- 支持在结果中展示所属知识库与路径信息。

### 7.1.5 访问与 SEO 基础能力

- 前台公开文章页应可直接访问。
- 支持基础 SEO 元信息。
- 支持 `sitemap` 与 `robots` 的后续扩展规划。

## 7.2 后台管理

### 7.2.1 后台登录

- 后台仅有一个管理员账号。
- 默认初始化账号密码为 `admin / admin`。
- 首次进入后台后应支持主动修改用户名与密码。

### 7.2.2 账号信息修改

- 支持修改用户名。
- 支持修改密码。
- 支持旧密码校验。
- 支持密码强度约束。

### 7.2.3 文章上传

- 仅支持上传 Markdown 文件与图片文件。
- 不提供在线编辑器。
- 支持选择目标知识库目录。
- 支持一次上传单个文章及其关联资源。
- 支持校验非法文件类型。

### 7.2.4 知识库树查看

- 后台可查看当前知识库树形结构。
- 可明确展示目录、Markdown 文件、资源目录。
- 前台树中隐藏 `resource`，后台树中可见以便管理员确认资源关系。

## 8. 内容组织规则

建议文件结构：

```text
content/
  knowledge-base-a/
    overview.md
    topic-1/
      article-a.md
      resource/
        image-1.png
        image-2.jpg
    topic-2/
      article-b.md
  knowledge-base-b/
    intro.md
```

规则说明：
- `content` 为知识内容根目录。
- 一级目录默认为一个知识库。
- 子目录用于形成树形结构。
- `.md` 文件为文章节点。
- `resource` 为文章所在目录的图片资源目录。
- 前台导航排除 `resource`。

## 9. 典型用户流程

### 9.1 访客浏览流程

1. 打开首页。
2. 选择一个知识库。
3. 在左侧树中展开目录。
4. 点击文章进入详情页。
5. 若目标不明确，则通过搜索直接定位文章。

### 9.2 管理员上传流程

1. 访问隐藏后台地址。
2. 使用管理员账号登录。
3. 进入上传页。
4. 选择目标知识库目录。
5. 上传 `.md` 文件与关联图片。
6. 系统完成存储并刷新知识库树。

## 10. 非功能要求

- 可维护性：内容基于文件系统组织，便于备份。
- 易迁移性：Markdown 为主格式，降低平台绑定。
- 安全性：后台仅单管理员可访问，必须具备基础登录保护。
- 性能：前台树导航与文章详情加载应保持轻量。
- 可扩展性：后续可扩展标签、专题、草稿、全文索引。

## 11. 风险与边界

- 默认 `admin / admin` 仅适合初始化场景，正式部署后应尽快修改。
- 仅依赖文件上传，后续若引入批量导入，需要额外定义冲突处理规则。
- 若 Markdown 图片路径写法不统一，需要在解析规则中进一步约束。
- 若知识库内容规模增大，搜索实现应预留索引升级空间。

## 12. 验收标准

- 前台无需登录即可浏览知识库与文章。
- 首页默认展示知识库集合，不是普通博客时间流。
- 左侧存在类 IDE 的树形导航。
- 可通过搜索栏检索并打开单篇文章。
- 文章 Markdown 图片可按同级 `resource` 目录正常解析。
- 前台导航中不展示 `resource` 文件夹。
- 后台不在前台公开导航中暴露。
- 后台仅支持单管理员登录，默认 `admin / admin`。
- 后台可修改用户名和密码。
- 后台支持 `md + 图片` 上传，不提供编辑页。
- 后台可查看当前知识库树形文件结构。
